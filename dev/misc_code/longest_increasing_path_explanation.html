üîç Longest Increasing Path in Matrix

Deep Dive with DFS + Memoization

üìã Algorithm Overview

The Longest Increasing Path in a Matrix problem asks: given a 2D matrix of integers, find the length of the longest path where each step moves to a neighboring cell (up, down, left, right) with a strictly greater value.

Key Insight: This is a classic DFS (Depth-First Search) problem with memoization. We explore all possible increasing paths from each cell and cache results to avoid redundant calculations.

The Strategy

  1. Try every cell as a starting point - We don't know where the longest path begins
  2. Use DFS to explore paths - From each cell, recursively explore neighbors with larger values
  3. Memoize results - Cache the longest path from each cell to avoid recalculating
  4. Track the maximum - Keep updating the global maximum path length

üíª Complete Code with Detailed Comments

from typing import List def longest_increasing_path(matrix: List[List[int]]) -> int: # Handle empty matrix edge case if not matrix: return 0 # Initialize result to track maximum path length res = 0 # Get matrix dimensions m, n = len(matrix), len(matrix[0]) # Create memoization table (initially all zeros) # memo[r][c] will store the longest path starting from (r, c) memo = [[0] * n for _ in range(m)] # Try each cell as a potential starting point for r in range(m): for c in range(n): # Find longest path from this cell and update global max res = max(res, dfs(r, c, matrix, memo)) return res def dfs(r: int, c: int, matrix: List[List[int]], memo: List[List[int]]) -> int: # If we've already computed the longest path from this cell, return it if memo[r][c] != 0: return memo[r][c] # Base case: minimum path length is 1 (the cell itself) max_path = 1 # Four directions: up, down, left, right dirs = [(-1, 0), (1, 0), (0, -1), (0, 1)] # Try moving to each neighboring cell for d in dirs: next_r, next_c = r + d[0], c + d[1] # Only move if neighbor is valid and has a larger value if is_within_bounds(next_r, next_c, matrix) and matrix[next_r][next_c] > matrix[r][c]: # Path length = 1 (current cell) + longest path from neighbor max_path = max(max_path, 1 + dfs(next_r, next_c, matrix, memo)) # Cache the result for this cell memo[r][c] = max_path return max_path def is_within_bounds(r: int, c: int, matrix: List[List[int]]) -> bool: # Check if position (r, c) is within matrix bounds return 0 <= r < len(matrix) and 0 <= c < len(matrix[0])

Memoization Saves Computation

Without memoization: If we started from different cells, we'd recalculate paths multiple times.

With memoization: Once we compute the longest path from cell (0,1) = 2, we never need to compute it again. Any other path that reaches 2 can simply add 1 + memo[0][1].

For this 4√ó4 matrix, memoization reduces time complexity from potentially O(2^(m*n)) to O(m*n)!

‚ö° Complexity Analysis

Time Complexity

O(m √ó n)

Why?

  • We have m √ó n cells in the matrix
  • Each cell is visited exactly once due to memoization
  • For each cell, we check 4 neighbors (constant time)
  • Without memoization, worst case would be O(2^(m√ón)) - exponential!
Key Insight: Memoization transforms this from an exponential problem to a linear one!

Space Complexity

O(m √ó n)

Breakdown:

  • Memoization table: O(m √ó n) space to store results for each cell
  • Recursion stack: O(m √ó n) in worst case (imagine a path that visits every cell)
  • Total: O(m √ó n) + O(m √ó n) = O(m √ó n)

üí° Key Insights & Takeaways

Why This Algorithm Works

  1. Optimal Substructure: The longest path from a cell depends on the longest paths from its valid neighbors. If we know the longest path from each neighbor, we can compute our cell's longest path.
  2. Top-Down DP: We start from each cell and recursively explore, caching results as we go.
  3. No Cycles: Since we only move to strictly greater values, we can never revisit a cell (no infinite loops!).

When to Use This Pattern

  • Finding paths in a matrix with constraints
  • Problems with "longest/shortest" paths
  • When subproblems overlap (same cell visited from different paths)
  • Graph problems on implicit graphs (matrix as graph)

Common Variations

  • Longest Decreasing Path: Change condition to matrix[next] < matrix[curr]
  • Sum Instead of Length: Return sum of values instead of count
  • Different Directions: Add diagonal movement (8 directions)
  • Weighted Graph: Each cell has a cost/weight

Optimization Tips

  • Early termination: If a cell has no valid neighbors, immediately return 1
  • Sort by value: Process cells in increasing order of value (alternative approach)
  • Space optimization: Can't really reduce space much - we need the memo table

‚ùå Common Mistakes to Avoid

1. Forgetting to Initialize Memo

Always create the memo table before starting DFS. Without it, you'll get exponential time complexity.

2. Using >= Instead of >

The path must be strictly increasing. Using >= would allow equal values, which violates the problem constraints.

3. Not Checking Bounds

Always validate that next_r and next_c are within matrix bounds before accessing matrix[next_r][next_c].

4. Forgetting Base Case

Every cell has at least a path length of 1 (itself). Don't forget to initialize max_path = 1.

5. Not Memoizing Before Returning

Make sure to store the result in memo[r][c] before returning. Otherwise, future calls won't benefit from caching.
