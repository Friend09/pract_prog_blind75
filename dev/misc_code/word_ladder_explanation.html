üîÑ Shortest Transformation Sequence

Bidirectional BFS Word Ladder Algorithm

Problem Statement

Given two words (start and end) and a dictionary containing an array of words, return the length of the shortest transformation sequence to transform start to end.

A transformation sequence is a series of words in which:

  • Each word differs from the preceding word by exactly one letter
  • Each word in the sequence exists in the dictionary

If no such transformation sequence exists, return 0.

Example:
start = "red", end = "hit"
dictionary = ["red", "bed", "hat", "rod", "rad", "rat", "hit", "bad", "bat"]
Output: 5
Path: red ‚Üí bed ‚Üí bad ‚Üí bat ‚Üí hat ‚Üí hit

üß† Algorithm Overview

What is Bidirectional BFS?

This algorithm uses Bidirectional Breadth-First Search (BFS) - a powerful optimization where we simultaneously search from both the start word and the end word, meeting in the middle.

‚úÖ Bidirectional BFS

Searches from both ends simultaneously

Time: O(b^(d/2))

Much faster for large search spaces!

Standard BFS

Searches only from start to end

Time: O(b^d)

Explores many more nodes

Why is this faster?
If branching factor is b and distance is d:
‚Ä¢ Standard BFS explores: b^d nodes
‚Ä¢ Bidirectional BFS explores: 2 √ó b^(d/2) nodes

Example: If b=10 and d=6: Standard = 1,000,000 nodes vs Bidirectional = 2,000 nodes!

from collections import deque from typing import List def shortest_transformation_sequence_optimized( start: str, end: str, dictionary: List[str] ) -> int: # Convert dictionary list to set for O(1) lookup dictionary_set = set(dictionary) # Edge case: if start or end not in dictionary, no path exists if start not in dictionary_set or end not in dictionary_set: return 0 # Edge case: if start equals end, path length is 1 if start == end: return 1 # Initialize queue and visited set for forward search (from start) start_queue = deque([start]) start_visited = {start} # Initialize queue and visited set for backward search (from end) end_queue = deque([end]) end_visited = {end} # Track the number of levels explored from each direction level_start = level_end = 0 # Continue while both queues have words to explore while start_queue and end_queue: # Explore next level from start word level_start += 1 if explore_level( start_queue, start_visited, end_visited, dictionary_set ): # Searches met! Return total path length return level_start + level_end + 1 # Explore next level from end word level_end += 1 if explore_level( end_queue, end_visited, start_visited, dictionary_set ): # Searches met! Return total path length return level_start + level_end + 1 # If we exit the loop, no path exists return 0 def explore_level( queue, visited, other_visited, dictionary_set ) -> bool: # All lowercase letters for generating transformations lower_case_alphabet = 'abcdefghijklmnopqrstuvwxyz' # Process all words in the current level for _ in range(len(queue)): current_word = queue.popleft() # Try changing each position in the word for i in range(len(current_word)): # Try all 26 possible letters for c in lower_case_alphabet: # Generate new word by changing character at position i next_word = current_word[:i] + c + current_word[i + 1:] # Check if this word was visited by the OTHER search if next_word in other_visited: return True # Searches met! # If word is valid and not visited, add to queue if next_word in dictionary_set and next_word not in visited: visited.add(next_word) queue.append(next_word) # Searches haven't met yet return False

üí° Key Insights & Complexity

Time Complexity

O(N √ó L √ó 26) where:

  • N = number of words in dictionary
  • L = length of each word
  • 26 = trying all lowercase letters

For each word, we try changing each of its L positions to 26 possible letters. In the worst case, we might visit all N words in the dictionary.

Bidirectional optimization: Reduces the practical search space significantly! If the shortest path has length d, we explore roughly O(b^(d/2)) instead of O(b^d) nodes.

Space Complexity

O(N)

  • dictionary_set: O(N) to store all words
  • start_visited: O(N) in worst case
  • end_visited: O(N) in worst case
  • start_queue: O(N) in worst case
  • end_queue: O(N) in worst case
Why Convert Dictionary to Set?
Lists have O(N) lookup time, while sets have O(1) lookup time.
Since we check "if next_word in dictionary_set" millions of times,
this optimization is crucial for performance!
Why Bidirectional BFS?
Imagine searching for a path of length 10 with branching factor 5:
‚Ä¢ Standard BFS: 5^10 = 9,765,625 nodes explored
‚Ä¢ Bidirectional BFS: 2 √ó 5^5 = 6,250 nodes explored
That's a 1,562x speedup!
When Do the Searches Meet?
When we're processing a word from one direction and generate a neighbor
that has already been visited by the OTHER direction.
At that moment, we've found the shortest path!

üéì Summary

Algorithm Steps:

  1. Initialize two BFS searches from start and end
  2. Alternate between exploring levels from each direction
  3. For each word, generate all possible one-letter transformations
  4. Check if any generated word was visited by the OTHER search
  5. If searches meet, return total distance
  6. If queues become empty, no path exists

Why This Algorithm Works:

BFS guarantees we find the shortest path because it explores all words at distance d before exploring words at distance d+1. By searching from both ends, we reduce the search space dramatically and meet in the middle, giving us the optimal solution faster.
